DAW05.- Solución a la tarea.

Solución de la tarea para DAW05.

  1. Configurar dos servidores BIND:
    1. Uno como servidor maestro con nombre DNS ns1.empresa-tarea-daw05.local en la IP 192.168.200.250.
      • Instalar un servidor DNS BIND (bind9). 
        apt-get update
apt-get upgrade
apt-get install bind9 bind9utils
      • Añadir al final del fichero de configuración /etc/bind/named.conf.local lo siguiente: 
        //zonas creadas tipo master
zone "empresa-tarea-daw05.local" {
        type master;
        file "/var/lib/bind/master/db.empresa-tarea-daw05.local.hosts";
};
      • Así el servidor sirve el dominio empresa-tarea.daw05.local como master, y la zona se guarda en el fichero /var/lib/bind/master/db.empresa-tarea-daw05.local.hosts
      • Configurar el fichero /var/lib/bind/master/db.empresa-tarea-daw05.local.hosts para agregar el registro de recurso tipo A: 
        ;
; BIND Database file for empresa-tarea-daw05.local zone 
;
@     IN SOA      ns1.empresa-tarea-daw05.local.  hostmaster.empresa-tarea-daw05.local. (
2011091601; serial number
3600; refresh
600; retry
1209600; expire
3600) ; default TTL
;
@     NS         ns1.empresa-tarea-daw05.local. 
;
@     A          192.168.200.250
ns1   A          192.168.200.250
www   A          192.168.200.250
      • NOTA: puedes realizar la siguiente consulta: named-checkzone empresa-tarea-daw05.local /var/lib/bind/master/db.empresa-tarea-daw05.local.hosts para verificar la correcta configuración de la zona.
        • Recargar el servidor con el comando systemctl restart bind9 ó service bind9 reload ó /etc/init.d/bind9 reload
    2. Otro como servidor esclavo con nombre DNS ns2.empresa-tarea-daw05.local en la IP 192.168.200.249.
      • En el servidor maestro:
        • Añadir en el archivo /var/lib/bind/master/db.empresa-tarea-daw05.local.hosts las líneas: 
          @     NS         ns2.empresa-tarea-daw05.local.
ns2   A          192.168.200.249
        • Modificar el número de serie de 2011091601 a 2011091602
        • Recargar el servidor systemctl restart bind9 ó service bind9 reload ó /etc/init.d/bind9 reload
      • Para el servidor esclavo (host con IP 192.168.200.249):
        • Instalar un servidor DNS BIND (bind9). 
          apt-get update
apt-get upgrade
apt-get install bind9 bind9utils
        • Añadir al final del fichero de configuración /etc/bind/named.conf.local lo siguiente: 
          zone "empresa-tarea-daw05.local" {
        type slave;
        masters {
                192.168.200.250;
                };
        file "/var/lib/bind/slave/db.empresa-tarea-daw05.local.hosts";
};
        • Recargar el servidor con el comando: systemctl restart bind9 ó service bind9 reload ó /etc/init.d/bind9 reload
    3. Con la posibilidad de transferencia de zona entre ellos.
      • En el servidor maestro configuras la sección correspondiente al servidor master en el fichero /etc/bind/named.conf.local: 
        //zonas creadas tipo master
     zone "empresa-tarea-daw05.local" {
          type master;
          file "/var/lib/bind/db.empresa-tarea-daw05.local.hosts";
          allow-transfer{192.168.200.249;};
          // notify yes;
     };
      • Recargar el servidor con el comando: systemctl restart bind9 ó service bind9 reload ó /etc/init.d/bind9 reload
    4. Donde la zona del servidor maestro contiene los siguientes registros de recursos:
      • mail que apunta a la IP 192.168.100.249
      • redmine que apunta al nombre DNS www.empresa-tarea-daw05.local
      • aplicacion-web que es un alias de redmine

        Modifica el fichero /var/lib/bind/db.empresa-tarea-daw05.local.hosts. con el siguiente contenido:

        ;
; BIND Database file for empresa-tarea-daw05.local zone 
;
@	IN	SOA	ns1.empresa-tarea-daw05.local. hostmaster.empresa-tarea-daw05.local. (
			2011091603
			3600
			600
			1209600
			3600 )
;
@               NS          ns1.empresa-tarea-daw05.local.
@               NS          ns2.empresa-tarea-daw05.local. 
@               MX    10    mail.empresa-tarea-daw05.local.
;
@               A          192.168.200.250
ns1             A          192.168.200.250
www             A          192.168.200.250
ns2             A          192.168.200.249
mail            A          192.168.100.249
redmine         CNAME      www
aplicacion-web  CNAME      redmine

        NOTA: recordar modificar el número de serie.

      • Recargar el servidor con el comando: systemctl restart bind9 ó service bind9 reload ó /etc/init.d/bind9 reload
    5. Donde la transferencia de zona se realiza de forma automática cuando se modifica la zona en el servidor maestro.
      • Habilitar 
        notify yes
        → En el servidor maestro configurar la sección correspondiente al servidor master en el fichero /etc/bind/named.conf.local:
        //zonas creadas tipo master
zone "empresa-tarea-daw05.local" {
        type master;
        file "/var/lib/bind/master/db.empresa-tarea-daw05.local.hosts";
        allow-transfer {192.168.200.249;};
        notify yes;
};
      • Recargar el servidor con el comando: systemctl restart bind9 ó service bind9 reload ó /etc/init.d/bind9 reload

    2. Una estructura básica para atender a una unidad organizativa que contenga el departamento de atención al cliente.
      • Crea la estructura básica del dominio LDAP mediante la ejecución de un fichero estructura-basica-tarea-daw05.ldif. Tener en cuenta una nota que dice que el departamento de atención al cliente será considerado como una unidad organizativa de nombre: operador.

        Crea la estructura básica del dominio LDAP mediante la ejecución de un fichero estructura-basica-tarea-daw05.ldif

        # Objetos raiz del dominio
dn: dc=empresa-tarea-daw05,dc=local
objectClass: top
objectClass: dcObject
objectclass: organization
o: empresa-tarea-daw05.local
dc: empresa-tarea-daw05
description:  Raiz de dominio
# Usuarios
dn: ou=operador,dc=empresa-tarea-daw05,dc=local
objectClass: organizationalUnit
ou: operador
        NOTA: revisa el archivo /etc/ldap/slapd.d/cn\=config/olcDatabase\=\{1\}hdb.ldif

        Ejecutar:

        # ldapadd -x -D cn=admin,dc=empresa-tarea-daw05,dc=local -w admin -f estructura-basica-tarea-daw05.ldif 
adding new entry "dc=empresa-tarea-daw05,dc=local"
adding new entry "ou=operador,dc=empresa-tarea-daw05,dc=local"
